What this is

A daily institutional-grade cross-asset read covering crypto, US equities, Indonesia (IDX), China & Hong Kong, and macro. Each note answers: what's happening, what's driving it, what to watch, and where conviction sits.

The framework is CFA-curriculum-grounded — scenarios with explicit probabilities, position ratings with conviction tags, risk-management front and centre. The tone is institutional; the audience is personal.

Methodology — how each daily note is produced

  1. Live data pull — fresh crypto + derivatives feed (BTC, ETH, SOL, XRP perpetuals + gold) with open interest, basis, and volume.
  2. Cross-asset web research — current US equities, macro/rates, oil, China/HK, IDX headlines and corporate-action calendar.
  3. Synthesis — CFA-framed: bottom line, cross-asset snapshot, drivers, technicals, capital rotation, Bull/Base/Bear scenarios per domain.
  4. Positions — 8–12 named Buy/Hold/Sell calls with conviction, time horizon, thesis, and the specific risk that breaks each one.
  5. Archive on publish — every note saved unedited at /archive/ so it ages publicly.

Position framework

Each position in the daily focus list carries four required tags:

  • Buy Hold Sell — directional view (or AVOID).
  • Conviction: Low / Medium / High — how strong the read is. "High" is rare on purpose.
  • Time horizon: typically 1–3m, 3–6m, 6–12m, or 12–24m — set explicitly so you can judge it later.
  • Thesis & key risk: a one-line "why I think this" + the single thing that would falsify it. If I can't articulate the risk, the call isn't ready.

Cap-tier coverage: mega- and large-caps for index-anchored exposure; named mid-cap picks when a clear catalyst or sentiment trigger warrants — each with thesis, risk and horizon. ETF / index proxies (e.g. Russell 2000 IWM, HSCEI) are still used for true small-cap exposure when individual-name accuracy would be a stretch.

Honesty rules I hold myself to

What I won't do

  • Quietly edit or delete past notes — they all live in /archive/.
  • Fabricate names without a real current catalyst. Mid-cap picks need defensible sentiment/news basis; deep small-caps stay as ETF proxies.
  • Label feed-derived metrics as "live" when they're authored — the page distinguishes live (browser-updated) vs authored (as-of timestamp).
  • Hide widget failures — if a panel breaks, I either fix it or remove it; the page degrades gracefully.
  • Frame this as financial advice. It is not.

What's live vs authored

  • Live (browser-updated): ticker tape, crypto price rows, crypto leaderboard + market-cap buckets, stock-mover panels.
  • Authored (as of feed timestamp shown): bottom line, drivers, scenarios, positions, technical levels, derivatives positioning, funding read.

Cadence

The note refreshes daily (on first app launch each day, when the auto-publish pipeline is working — manual updates fill the gap when it's not). Each refresh writes a new dated archive entry. Live prices on the page update continuously while it's open.
